다음 요구사항에 맞춰 Nginx 설정을 최적화하는 완전한 가이드를 만들어줘.

[목표]
- React 정적 파일(JS/CSS/폰트)을 클라이언트에게 전송할 때 트래픽을 최대한 줄인다.
- gzip 및 brotli 압축 설정을 적용한다.
- 캐시 설정(Cache-Control, ETag, expires)도 함께 최적화한다.
- HTML, JS, CSS, JSON, SVG, 폰트 등을 모두 압축 대상에 포함한다.
- JPG/PNG는 gzip으로 압축되지 않으므로 별도 이미지 최적화 전략을 포함한다.

[결과물 형태]
1. Nginx 설정 예시(server 블록 전체)
2. gzip 최적 설정
3. brotli 최적 설정
4. React build(dist 또는 build 폴더)와 Nginx 연동법
5. 캐싱 전략 (immutable, max-age, static assets)
6. 이미지 최적화 전략(WebP 변환 또는 CDN)
7. 각 항목이 왜 필요한지에 대한 기술적 설명
8. 내 환경에 바로 붙여 넣을 수 있는 설정 파일 형태로 출력

[내 환경 상황]
- React 프론트엔드는 build/ 폴더에 결과물이 들어있다.
- Nginx가 정적 파일을 서빙하고 있다.
- API 서버는 Gunicorn + Flask 조합이다.
- 정적 파일 압축은 반드시 Nginx에서 하길 원한다.

위 조건으로 최적화 문서를 만들어줘.
