NeuroNova 프로젝트 전체 검토
1. 프로젝트 개요
NeuroNova (Neurology + Nova)는 뇌종양 진단을 위한 임상 의사결정 지원 시스템(CDSS)입니다. 핵심 가치:


2. 시스템 아키텍처
[Flutter App (환자)]    [React Web (의료진)]
         ↓                      ↓
    [Nginx + Gunicorn]
         ↓
    [Django 메인 서버] ←→ [Flask AI 추론] + [Orthanc DICOM]
         ↓
  [MySQL + Redis]



3. 데이터베이스 설계
EMR 영역 (OpenEMR 호환)
Patient: 환자 기본 정보 (pid, 이름, 생년월일, 보험번호)
Encounter: 진료 세션 (날짜, 사유, 상태)
FormSOAP: SOAP 차트 (Subjective, Objective, Assessment, Plan)
FormVitals: 활력 징후 (혈압, 체온, BMI 자동 계산)
MergedDocument: 통합 문서 (참조 ID로 원본 데이터 연결)
Custom 영역 (NeuroNova Core)
Doctor: 의사 정보 (면허번호, 전문분야)
PatientDoctor: 환자-의사 N:M 관계
Appointment: 예약 관리 (상태, 방문 유형)
PatientPredictionResult: AI 진단 결과 (핵심)
모델명/버전, Orthanc UID, 예측 클래스, 신뢰도
XAI 이미지 경로, 특성 중요도
Human-in-the-loop: 의사 피드백 (CORRECT, INCORRECT, AMBIGUOUS)
Prescription: 처방전
NotificationLog: 알림 기록

RBAC 권한 정책
역할	권한	특이사항
Admin	All (Hard Delete 가능)	시스템 관리
Doctor	CRU (Delete 불가)	Soft Delete만 가능 (의료 무결성)
Nurse	R (EMR), C/U (Vitals)	진료기록 수정 불가
Patient	R (Self), C/U (Appointment)	본인 정보만 조회
